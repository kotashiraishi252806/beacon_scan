## 4. 追加要望①：選択方法によらず、測定中の並び替え動作が同じであることの確認

### 内容

「選択したAPを測定」ボタンでの測定時、測定中のAPが一覧の一番上に来て、測定完了後に一番下へ移動する動き（`onApStart` / `onApFinished`、MainActivity.kt 437〜451行目）が、「全て選択」ボタンで選んだ場合だけでなく、チェックボックスを個別にタップして選んだ場合でも同じように起きることを確認してください。

### 補足

コードを確認した限り、選択方法（「全て選択」ボタン／個別チェックボックス）に関わらず、最終的に `adapter.getSelectedItems()`（選択中のbssid集合）を対象に同じ処理が走るため、実装上は既に選択方法によらず同一の動作になっているはずです。実機での動作確認のみで問題ないと想定していますが、もし選択方法によって挙動差がある場合はご報告ください。

## 5. 追加要望②：測定状態に応じたリスト項目の色分け表示

### 内容

「選択したAPを測定」ボタンによる**手動選択フロー**において、一覧の各AP項目を以下のように色分けし、未測定・測定中・測定済みを視覚的に区別できるようにしてください。

※「手動選択フロー」とは、「全て選択」ボタンで一括選択した場合・チェックボックスで個別に選択した場合の**両方**を含みます（測定自体はどちらも同じ「選択したAPを測定」ボタンから開始するため）。自動スキャン（switchAutoScan ON）のときは今回の色分けの対象外とし、現状の表示のままとしてください。

| 状態 | 表示 |
|---|---|
| 未測定（まだ測定順が回ってきていない） | 現状の見た目のまま（変更なし） |
| 測定中（現在接続測定を行っている1件） | 現状の見た目のまま（変更なし） |
| 測定済み（測定が完了した項目） | 暗めのグレーを50%透過で重ねた表示にする |

補足：測定中は選択モードが解除されチェックボックスが非表示になる（`btnMeasureSelected.setOnClickListener` 内、415〜418行目）ため、測定開始後はこの色分けが進捗状況を伝える唯一の手がかりになります。

### 実装案（参考）

`AccessPoint.supplicantFinalState` は初期値が `"NOT_MEASURED"` で、測定完了時（`measureSupplicantForAp()` 内）に別の値へ更新されます。この値を使って、`ApAdapter.onBindViewHolder()`（MainActivity.kt 1327行目〜）で以下のような判定を行うことで実現できると考えられます。

```kotlin
if (ap.supplicantFinalState != "NOT_MEASURED") {
    // 測定済み：暗いグレーを50%透過で重ねる
} else {
    // 未測定・測定中：現状のまま
}
```

「測定中」の1件だけを別扱いする必要はなく、`supplicantFinalState` が更新されるのは測定完了後（`onApFinished` のタイミング）のみのため、上記の判定だけで「未測定・測定中は現状の色／測定済みのみグレー」という要望を満たせるはずです。実装しやすい方法であれば上記に限らず問題ありません。

### 適用範囲

自動スキャン（switchAutoScan ON）時は対象外です。現状「並び替えなし」の実装のままで問題ありません。今回の色分けは手動選択フローのみに適用してください。
