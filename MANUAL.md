# BeaconScan 操作マニュアル

## 概要

周辺の無線LANアクセスポイント（AP）をスキャンし、測定データをサーバーへ送信するAndroidアプリです。手動・自動の2種類の測定モードと、オフライン時のデータ保持機能を備えています。

---

## 画面構成

| UI要素 | 説明 |
|---|---|
| 送信先URL入力欄 | データ送信先のサーバーURLを入力 |
| スキャンボタン | 手動で1回測定を実行（自動スキャン中は無効） |
| 自動スキャントグル | 10秒間隔の自動測定をON/OFF |
| 自動測定ステータス | 自動スキャン中のみ表示。「自動測定中」または「測定停止中」を表示 |
| 一時停止ボタン | 自動スキャン中のみ表示。スケジューリングを一時停止 |
| 送信するトグル | スキャン後に自動送信するかどうかを切り替え |
| 今回検出・合計表示 | スキャン完了後に「今回検出: N件 / 未送信データ合計: N件」を表示 |
| 一覧ボタン | 直近のスキャン結果一覧を表示／非表示（送信完了後は非表示） |
| 未送信データを送信ボタン | ローカルに保持している未送信データを手動送信 |
| 破棄ボタン | 破棄するデータをラベル単位で選択して削除 |
| 未送信スキャン数表示 | 現在ローカルに保持しているスキャン数とラベル内訳を表示 |

---

## 基本操作

### 送信先URLの設定

アプリ起動後、最上部の入力欄にデータ送信先のサーバーURLを入力します。入力内容はアプリ終了後も保持されます。

---

### 手動測定

**スキャンボタン**を押すと、その時点の周辺APを1回スキャンします。

- WiFiがオフの場合はスキャンできません（トーストでエラーを表示）
- 自動スキャン中はスキャンボタンが無効になります
- スキャン完了後、「今回検出: N件 / 未送信データ合計: N件」が表示されます
- 手動測定時はラベルなし（空）でデータが記録されます

---

### 自動測定

**自動スキャントグル**をONにすると、10秒間隔で自動的にスキャンを繰り返します。

- 自動スキャン中はスキャンボタンが無効になります
- 画面上部に「自動測定中」のステータスが表示されます

#### セッションの概念

自動スキャンのONからOFFまでを1つの**セッション**として扱います。セッション内のすべてのスキャンデータには同一のラベルが付与されます。

#### 一時停止

自動スキャン中に**一時停止ボタン**を押すと、スキャンのスケジューリングを停止しますが、セッションは継続します。**再開ボタン**を押すとスキャンが再開されます。

- ステータス表示が「測定停止中」に変わります
- 一時停止中のデータはセッションと同じラベルとして扱われます
- 例：13:00〜14:00のセッション中、13:30〜13:35を一時停止しても、前後のデータは同一セッションのデータです

#### ラベルの自動生成

自動スキャントグルをOFFにすると、ラベル設定ダイアログが表示されます。

```
┌─────────────────────────────────────┐
│ ラベルを設定しますか？               │
│                                     │
│ [2026/06/11 13:00~14:00           ] │ ← 編集可能
│                                     │
│      [そのまま保存]  [変更して保存]  │
└─────────────────────────────────────┘
```

- **そのまま保存**：自動生成されたラベル（開始〜終了時刻）で保存
- **変更して保存**：入力欄を編集した内容で保存

---

## データの送信

### 自動送信（送信するトグルON）

スキャン完了後、自動的にサーバーへ送信します。送信完了後はスキャン結果の一覧ボタンが非表示になります。送信失敗した場合はデータをローカルに保持し、次回スキャン時に再送を試みます。

### 手動送信（送信するトグルOFF）

データはすべてローカルに保持されます。任意のタイミングで**未送信データを送信ボタン**を押すことで、保持しているすべてのデータを送信できます。

- 送信はスキャン1件ずつ順に行われます
- 送信成功後、未送信スキャン数は0にリセットされ、一覧ボタンが非表示になります
- 途中で失敗した場合は、失敗した件数が残ります

### 未送信データの破棄

**破棄ボタン**を押すと、ラベル単位でデータを選択できるダイアログが表示されます。

```
┌──────────────────────────────────────────┐
│ 破棄するデータを選択                      │
│                                          │
│ ☑ 2026/06/11 13:00~14:00 (10回)         │
│ ☐ 2026/06/11 15:00~16:00 (8回)          │
│                                          │
│              [キャンセル]  [破棄する]     │
└──────────────────────────────────────────┘
```

- チェックしたラベルのデータのみを削除します
- この操作は取り消せません

---

## ラベルとデータ管理

ラベルはセッション（自動スキャンのON〜OFF）単位で自動生成されます。異なるセッションには異なるラベルが付くため、複数セッションのデータをローカルに溜め込んだ場合でも、ラベルで区別できます。

**未送信スキャン数表示**の例：
```
未送信スキャン数: 18回
  2026/06/11 13:00~14:00: 10回
  2026/06/11 15:00~16:00: 8回
```

送信時はラベルに関わらず全件を一括送信します。サーバー側でラベルによるフィルタリングやグループ化が可能です。

---

## 送信データの形式

スキャン1件ごとに以下のJSONがサーバーへ送信されます。

```json
{
  "scan_id": "uuid",
  "device_id": "uuid",
  "device": {
    "manufacturer": "Google",
    "model": "Pixel 8",
    "android_api": 34
  },
  "scanned_at": "2026-06-11T13:05:00",
  "label": "2026/06/11 13:00~14:00",
  "location": {
    "latitude": 35.6895,
    "longitude": 139.6917,
    "accuracy_m": 5.2
  },
  "access_points": [
    {
      "bssid": "AA:BB:CC:DD:EE:FF",
      "oui": "AA:BB:CC",
      "ssids": ["MyNetwork"],
      "rssi_dbm": -55,
      "frequency_mhz": 5180,
      "band": "5GHz",
      "channel_width_mhz": 80,
      "wifi_standard": "802.11ax",
      "wifi_standard_code": 6,
      "security": "WPA3",
      "capabilities_raw": "[WPA3-SAE-CCMP][ESS]"
    }
  ]
}
```

---

## 注意事項

- スキャンにはAndroidの位置情報権限（ACCESS_FINE_LOCATION）が必要です
- WiFiがオフの場合はスキャンできません
- `wifi_standard` はAndroid 11（API Level 30）以上で正確な値が取得できます
- `scanned_at` はISO 8601形式（`yyyy-MM-dd'T'HH:mm:ss`）で端末のタイムゾーンで記録されます（GMT+9の場合はJST）
